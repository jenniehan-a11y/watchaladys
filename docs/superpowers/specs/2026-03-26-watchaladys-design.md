# watchaladys - 맛집 저장 & 공유 웹사이트 디자인 스펙

## Context

전국 맛집을 "지역 → 동네 → 가게명" 구조로 저장하고, URL을 아는 친구들과 공유할 수 있는 웹사이트.
빈티지 감성(진한 초록+주황+라벤더)의 디자인으로, 메인 화면에 소주병 일러스트를 배치하고 클릭하면 맛집 목록으로 진입하는 구조.

---

## 기술 스택

- **프레임워크**: Next.js (App Router)
- **스타일링**: Tailwind CSS
- **DB/백엔드**: Supabase (PostgreSQL + JS SDK)
- **배포**: Vercel
- **프로젝트 위치**: `~/watchaladys/` (jennieblog와 완전 독립)

---

## 데이터 구조

### `restaurants` 테이블 (Supabase PostgreSQL)

| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid (PK, auto) | 자동 생성 |
| name | text (NOT NULL) | 가게명 |
| region | text (NOT NULL) | 지역 (서울, 부산, 제주 등 - 자유 입력) |
| neighborhood | text (NOT NULL) | 동네 (합정역, 성수 등 - 자유 입력) |
| category | text (NOT NULL) | 카테고리 (한식, 일식, 양식, 카페, 술집 등 - 자유 입력) |
| status | text (NOT NULL) | 'want_to_go' 또는 'visited' |
| naver_map_url | text (nullable) | 네이버 지도 링크 |
| instagram_url | text (nullable) | 인스타그램 링크 |
| memo | text (nullable) | 간단 메모 |
| created_at | timestamptz | 생성일 (auto) |
| updated_at | timestamptz | 수정일 (auto) |

### RLS 정책
- 인증 없음: anon key로 모든 CRUD 허용
- URL을 아는 사람이면 누구나 추가/수정/삭제 가능

## 페이지 구조

### 1. `/` - 메인 페이지
- 진한 초록 배경 전체 화면
- 가운데 소주병 SVG 일러스트 (라벤더 색상 병 + 주황 라벨)
- 소주병 클릭 또는 탭 → `/restaurants`로 이동
- 라벨에 사이트명 "watchaladys" 표시
- 별도 네비게이션 없음 (소주병이 유일한 진입점)

### 2. `/restaurants` - 맛집 목록 페이지
- **탭**: 가고싶다 / 다녀왔다 (상단 탭으로 전환)
- **필터**: 지역, 동네, 카테고리 드롭다운 (기존 데이터에서 동적 추출)
- **맛집 카드 리스트**:
  - 카드에 표시: 가게명, 동네, 카테고리
  - 카드 클릭/탭 → 상세 영역 확장 (네이버지도/인스타 버튼, 메모)
  - 네이버 지도 버튼: naver_map_url이 있으면 활성, 없으면 비활성
  - 인스타그램 버튼: instagram_url이 있으면 활성, 없으면 비활성
- **+ 버튼**: 우하단 FAB → `/restaurants/new`로 이동
- **수정/삭제**: 카드 상세에서 수정/삭제 버튼

### 3. `/restaurants/new` - 맛집 추가 페이지
- **입력 폼**:
  - 가게명 (필수, 텍스트)
  - 지역 (필수, 텍스트 - 기존 값 자동완성 제안)
  - 동네 (필수, 텍스트 - 기존 값 자동완성 제안)
  - 카테고리 (필수, 텍스트 - 기존 값 자동완성 제안)
  - 상태 (필수, 가고싶다/다녀왔다 토글)
  - 네이버 지도 링크 (선택)
  - 인스타그램 링크 (선택)
  - 메모 (선택)
- **저장** → `/restaurants`로 이동

### 4. `/restaurants/[id]/edit` - 맛집 수정 페이지
- 추가 페이지와 동일한 폼, 기존 데이터 프리필
- 상태 변경 가능 (가고싶다 ↔ 다녀왔다)

## 디자인 시스템

### 컬러 팔레트 (빈티지 + 대담한 색감)
레퍼런스: HABIBI(초록+빨강), OENO(라벤더+퍼플블루+다크브라운), 와인병(초록+주황+라벤더)
- **메인 배경**: 진한 초록 `#2D5016`
- **주황/레드**: `#E8652E` (라벨, 버튼, 액센트)
- **라벤더**: `#D4B8E0` (소주병 색상, 카드 배경)
- **퍼플블루**: `#6B7FD7` (땡땡이 패턴, 아이콘, 포인트)
- **다크브라운**: `#3D1A1A` (타이틀, 강조 텍스트 - OENO 참고)
- **빨강**: `#C42B2B` (삭제 버튼, 경고, 포인트)
- **오프화이트**: `#F5F0E8` (텍스트, 밝은 배경)
- **검정**: `#1A1A1A` (라벨 위 텍스트)
- 페이지별/카드별로 다양한 색상을 적극적으로 활용 (단조롭지 않게)

### 소주병 일러스트
- SVG/CSS로 제작 — 심플한 실루엣/형태만 (디테일한 일러스트 불필요)
- 라벤더 색상 병 실루엣 + 주황 라벨 영역 + 검정 텍스트
- 라벨에 "watchaladys" 표기
- **폴카닷(땡땡이) 패턴**: 라벨 위에 퍼플블루 계열의 큰 동그라미 포인트 배치 (Bornholms Kildevand 병 참고). 단조로운 색상 위에 귀여운 포인트 역할

### 타이포그래피
- 한글: Pretendard 또는 Noto Serif KR (빈티지 느낌)
- 라벨/영문: 세리프 계열

### 카드 스타일
- 둥근 모서리
- 주황 또는 오프화이트 배경
- 빈티지 라벨 느낌의 보더/장식

## 권한 모델

- 인증/로그인 없음
- Supabase anon key로 접근
- URL을 아는 모든 사람이 CRUD 가능
- RLS: `true` 정책으로 전체 공개

## 미래 확장 (현재 구현하지 않음)

- 같은 동네 맛집 추천
- 비슷한 카테고리 추천
- 최근 추가된 맛집 노출

---

## 검증 방법

1. `npm run dev`로 로컬 실행
2. 메인 페이지에서 소주병 클릭 → 맛집 목록 이동 확인
3. 맛집 추가 → DB 저장 확인 (Supabase 대시보드)
4. 가고싶다/다녀왔다 탭 전환 확인
5. 필터 (지역/동네/카테고리) 동작 확인
6. 수정/삭제 기능 확인
7. 네이버지도/인스타그램 링크 버튼 동작 확인
8. Vercel 배포 후 공유 URL로 접근 확인
